GitHub. HW_2
1. На локальном репозитории сделать ветки для:
- Postman --- $ git checkout -b Postman 
- Jmeter --- $ git checkout -b Jmeter 
- CheckLists --- $ git checkout -b CheckLists 
- Bag Reports --- $ git checkout -b BagReports
- SQL --- $ git checkout -b SQL 
- Charles --- $ git checkout -b Charles 
- Mobile testing --- $ git checkout -b MobileTesting
2. Запушить все ветки на внешний репозиторий --- git push --all
3. В ветке Bag Reports сделать текстовый документ со структурой баг репорта
	$ git touch BugReports_Structure.txt
	$ git vim BugReports_Structure.txt
		1. ID
		2. Summary
		3. Description
		4. Step to reproduce
		5. Reproducibility
		6. Severity
		7. Priority
		8. Symptom
		9. Workaround
		10. Comments
		11. Attachments
4. Запушить структуру багрепорта на внешний репозиторий
	$ git add .
	$ git commit -m "Add BugReports_Structure.txt file"
	$ git push
5. Вмержить ветку Bag Reports в Main
	$ git checkout main
	$ git merge BugReports
6. Запушить main на внешний репозиторий.
	$ git push -u origin main
7. В ветке CheckLists набросать структуру чек листа.
	$ vim touch CheckList_Structure.txt
	$ vim CheckList_Structure.txtЧеклист для тестирвоания локализации (спасибо хабр)
		1.Установка и удаление продукта при использовании локали, отличной от дефолтной.
		2. Настройка приложения на локали, отличной от дефолтной. Все изменения в настройках должны сохраняться и
			 использоваться.
		3.Приложение должно работать на окружении, локаль которого отличается от дефолтной локали нашего приложения.
		4.При запуске приложения на новой локали вам нужно проверить, что форматы даты, времени, спец.символы алфавита,
 			денежных валют, календари и прочие вещи, связанные с культурными особенностями можно переключать в
 			приложении без каких-либо ошибок (вручную или автоматически).
 			Как правило такие ошибки выявляются при первом запуске новой локали.
		5.Локализованный текст, изображения и прочие ресурсы не вшиты (hard-coded) в приложение.
		6.Документы (к примеру гайды, FAQ и т.д.) переключаются на нужный язык при смене локали.
8. Запушить структуру на внешний репозиторий
	$ git add .
	$ git commit -m "Add CheckList_Structure.txt file"
	$ git push
9. На внешнем репозитории сделать Pull Request ветки CheckLists в main
	1. Open https://github.com/MinoDentori/ksendzov_course_23
	2. Press Compare & Pull Request.
	3. Confirm merge.
	4. Create pull request.
